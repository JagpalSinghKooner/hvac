**Schema Corrections Needed:**
1. `problem.issues`: Must be `array of strings`, NOT `array of objects with title/description`
   - WRONG: `issues: [{title: "...", description: "..."}]`
   - RIGHT: `issues: ["Issue description 1", "Issue description 2", "Issue description 3"]`

2. `context`: Must be `string` directly, NOT `object with description key`
   - WRONG: `context: {description: "..."}`
   - RIGHT: `context: "Long string content here..."`

3. `rebateInfo`: Must be `string INSIDE savings object`, NOT separate top-level object
   - WRONG: 
     ```
     savings:
       headline: "..."
       description: "..."
     rebateInfo:
       primaryProgram: "..."
     ```
   - RIGHT:
     ```
     savings:
       headline: "..."
       description: "..."
       rebateInfo: "Save on Energy up to $6,200 via Guelph Hydro"
     ```

4. `proof`: Must be `object with testimonial/customerName/customerLocation`, NOT array
   - WRONG: `proof: [{type: "...", value: "..."}]`
   - RIGHT: Leave proof field EMPTY or use testimonial object structure (likely not needed for installation pages)

**Root Cause:**
Generated content using services schema mental model (with nested objects for issues) rather than service-city schema (simpler string-based structure). Service-city collection uses simpler schema for city-level content.
